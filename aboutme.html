The SoBot Project

The main objective of this project was to adapt the interface of a robot toy for a young girl, who has special needs. The robot itself is a dock-style base for an iOS device, called Romo, which was developed by Romotive. The interface is a suite of games that are appropriate for the user's level of development. Through these games, the interface, coupled with the Romo will aid and reinforce learning. By making the learning invisible, the user can learn and enjoy doing so.

The SoBot project is made up of two iOS applications, both built in Objective-C; the SoBotPad and the SoBotRobot. Each application is responsible for an aspect of the overall project. The two applications are able to communicate.

The SoBotRobot application is built on an iPod touch. This device then sits into the Romo base. The purpose of the SoBotRobot application is control of the Romo itself. The SoBotRobot application makes use of the Romo SDK in order to communicate with the Romo base. As well as this, the SoBotRobot application uses Multipeer Connectivity to connect to the SoBotPad application, for streaming text and video.

The SoBotPad application is the true interface of the project. While the SoBotRobot was mostly built on the Romo SDK, the SoBotPad application was built top to bottom. The SoBotPad is made up of a match the pairs game, a fill-in-the-blanks game and a remote control for the Romo. The design of the SoBotPad is the crucial element, as it was important to adhere to accessibility guidelines. The interface is controlled through a symbol-based language. The SoBotPad incorporates a wide variety of frameworks, which aid in various aspects of the application. The core technological features of the app include Multipeer Connectivity and text to speech.
